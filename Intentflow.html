What is LI.FI

The infrastructure layer
for cross-chain movement

Intents

Users Declare Intent,
Not Transactions

Instead of choosing a bridge, approving routers, and managing gas on multiple chains — users simply state what they want. The network handles the rest.

Traditional approach:
1. Bridge USDC Ethereum → Arbitrum
2. Wait 15 min for bridge confirmation
3. Swap USDC → ETH on Arbitrum DEX
4. 3× gas fees. 3× wallet prompts.
↓
With intents:
"fromToken": "USDC",
"fromChain": "Base",
"toToken": "ETH",
"toChain": "Arbitrum",
"amount": 100,
"recipient": "0xYourWallet"
// No routing. No bridge selection. No gas estimation.
🎯
Outcome-Based
Declare the desired output. Solvers compute the optimal execution path — no routing decisions required from you or your users.
🔒
Secured by Resource Locks
Funds are locked in escrow on the source chain. The solver only claims them after verified delivery on the destination.
⚡
Calls on Delivery
Attach calldata to your intent. After assets land, trigger DeFi actions automatically — stake, swap, deposit — all in one signature.

How It Works

From signature to settlement
in five steps

01
User Creates Intent
User specifies the desired output — token, amount, chain, recipient — and signs a StandardOrder. Funds are locked in InputSettlerEscrow.
02
Intent Broadcasted
The order server validates and co-signs the intent, then broadcasts to all connected solvers via WebSocket with full fill details.
03
Solvers Compete
Solvers match the order against their standing quotes. No live bidding — the server matches to the best pre-published price curve instantly.
04
Funds Delivered Cross-Chain
Winning solver calls fillOrderOutputs() on the destination chain, delivering the requested tokens from their own capital. User receives funds.
05
Verified Settlement
Oracle verifies delivery. Solver calls finalise() on InputSettler to reclaim locked source funds. Full cycle completes in under 2 minutes.
StandardOrder structure
struct StandardOrder {
address user;
uint256 nonce;
uint256 originChainId;
uint32 expires;
uint32 fillDeadline;
address inputOracle;
uint256[2][] inputs;
MandateOutput[] outputs;
}
Key insight: The outputs array holds MandateOutputs — each specifying what to deliver, where, and optional calldata to execute on arrival.
Capabilities

Everything builders need

🌐
Cross-Chain Interoperability
Route assets across all major EVM chains and Solana with a single intent object.
⚡
Fast Execution
Solver prefunding means users receive funds immediately. No waiting for cross-chain proofs.
🛡
MEV-Aware Routing
Standing-quote matching prevents frontrunning and sandwich attacks common in live auctions.
💧
Aggregated Liquidity
Solvers tap DEXes, CEXes, and private inventory — best fill regardless of source.
🔓
Permissionless Solvers
Anyone can become a solver. Open participation drives competitive pricing and resilience.
⛽
Gas Optimization
Optional gasless order submission. On-chain actions minimized to approval and deposit only.
